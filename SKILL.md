# 生鲜内容策划 + 即梦生成全流程（整合版）

用于一次性完成从商品信息到三条可交付短视频的全链路执行。

## 执行约束

- 全流程产出必须强绑定商品信息与商详内容，禁止泛化
- 必须产出 A/B/C 三套完整方案，且差异化明显
- 生成视频阶段只选“最推荐方案”执行，最终仅生成 1 条视频
- 每套必须标注适配场景与建议投放平台
- 若某套已建议平台，视频内容必须遵循该平台规则
- 分镜前置镜头不以商品主图复读为主，品牌与完整商品只在最后一镜集中露出
- 允许必要人物轻互动，但不得喧宾夺主
- 尽量不出现金额与价格相关表达
- 禁止以配料表/配料明细/成分列表作为创意或镜头重点
- 总时长约束：单条视频 `<=12秒`

## 输入参数规范

```
商品名称:
skuId:
商品品类:
商品主图目录（可选）:
商详页URL（可选）:
核心卖点（可选）:
辅助卖点（可选）:
目标用户:
用户需求:
期望风格（可选）:
目标时长（默认10-12秒）:
输出根目录（默认 ~/7freshVedioGenerate）:
目标分辨率（默认1080p）:
目标比例（默认9:16）:
```

## 目录与文件规则（必须执行）

1. 商品目录（策划产物）：
   - `最终商品目录 = 输出根目录/商品名称-skuId`（冲突自动递增）
2. 成片目录（视频产物）：
   - `最终成片目录 = 输出根目录/最终目录名-videos`（冲突自动递增）
3. 策划文件（6个）：
   - `最终目录名-商详卖点提取.md`
   - `最终目录名-切入点.md`
   - `最终目录名-创意结构.md`
   - `最终目录名-分镜脚本-A.md`
   - `最终目录名-分镜脚本-B.md`
   - `最终目录名-分镜脚本-C.md`
4. 视频文件（1个）：
   - `最终目录名-视频-推荐方案.mp4`
5. 执行记录（1个）：
   - `最终目录名-即梦生成记录.md`

## 总流程概览

```
0. 主图采集/复用校验
0.5 商详卖点提取（文本 + 图片文字）
1. 生成A/B/C三套切入点（场景+平台）
2. 生成A/B/C三套创意结构（场景+平台）
3. 生成A/B/C三套分镜脚本（三个文件）
4. 构建“推荐方案”即梦提示词
5. 即梦生成视频（仅推荐方案）
6. 下载命名归档 + 质量复核 + 回传
```

---

## 第0步：主图采集/复用校验

### 目标
确保后续创意有稳定视觉参考素材。

### 要点

- 若用户未提供主图：在 `7fresh.m.jd.com` 商详页采集
- 优先商详前两个楼层主图，不足再补充
- 统一命名：`最终目录名-1.jpg` ~ `最终目录名-5.jpg`
- 质量检查：可打开、JPG、建议 >100KB、至少3张

---

## 第0.5步：商详卖点提取（必须执行）

### 目标
形成后续策划的卖点事实底稿。

### 双通道提取

1. 页面文本通道：标题、详情模块、参数说明
2. 图片文字通道：轮播图/长图中的文案（等价 OCR 识别）

### 结构化输出

每条卖点记录：

```
卖点名称:
卖点类型:
卖点描述:
来源（文本/图片文字/双来源）:
证据片段:
可信度（高/中/低）:
```

### 分级

- 核心卖点 1-2 条
- 辅助卖点 2-4 条
- 去掉纯口号、不可验证描述、促销噪音

---

## 第1步：生成A/B/C三套切入点

### 目标
输出 3 套明显差异化切入点，并绑定场景与平台。

### 规则

- A：情绪驱动
- B：场景问题
- C：对比反差
- 可参考主图与商详图视觉线索
- 不把配料表作为创意主轴
- 不写价格/折扣导向表达

### 每套字段

```
方案编号（A/B/C）:
切入点名称:
方案定位:
适配场景:
适配平台:
目标人群:
核心痛点:
核心表达:
卖点-情绪映射:
场景映射:
开场话术:
收尾金句:
预期转化动作:
```

---

## 第2步：生成A/B/C三套创意结构

### 目标
将切入点转成可执行镜头节奏。

### 结构要求

- 每套 3-5 镜，10-12 秒
- 前1-4镜不露品牌、不长时间居中完整商品
- 第5镜（收尾镜）完成品牌与完整商品露出
- 每镜含：用户需求 + 商品卖点 + 镜头文案

### 平台适配规则（默认）

- 抖音：问题/反差开场，快节奏，CTA直接
- 小红书：生活感开场，质感细节，CTA偏收藏讨论
- 视频号：稳健叙事，信息完整，CTA偏关注转发

---

## 第3步：生成A/B/C三套分镜（3个文件）

### 输出顺序（必须）

- 先 A，再 B，再 C

### 文件名（固定）

- `最终目录名-分镜脚本-A.md`
- `最终目录名-分镜脚本-B.md`
- `最终目录名-分镜脚本-C.md`

### 推荐方案决策（新增，必须执行）

- 基于以下维度对 A/B/C 打分并排序：
  - 平台匹配度
  - 卖点清晰度
  - 场景代入感
  - 转化动作自然度
  - 风险合规性（价格/配料表/平台错配）
- 输出 `推荐方案编号`（A/B/C）与 `推荐理由`（2-4条）
- 第4步和第5步仅使用该推荐方案

### 推荐方案打分模板（带权重，必须执行）

评分采用 100 分制，按以下权重计算总分：

- 平台匹配度：40%
- 转化潜力：30%
- 合规性与执行稳定性：30%

各维度评分标准（每项 0-100 分）：

1. 平台匹配度（40%）
   - 开场钩子与平台习惯匹配
   - 文案语气与互动方式匹配
   - 节奏与信息密度匹配

2. 转化潜力（30%）
   - 卖点是否清晰、可感知
   - 场景是否可代入
   - CTA 是否自然且有效

3. 合规性与执行稳定性（30%）
   - 是否规避金额/价格表达
   - 是否规避配料表主体表达
   - 品牌露出时机与镜头约束是否稳定可执行

计算公式：

`总分 = 平台匹配度*0.40 + 转化潜力*0.30 + 合规稳定性*0.30`

打分记录模板（写入 `即梦生成记录.md`）：

```
推荐方案评分：
- 方案A：平台匹配度__ / 转化潜力__ / 合规稳定性__ / 总分__
- 方案B：平台匹配度__ / 转化潜力__ / 合规稳定性__ / 总分__
- 方案C：平台匹配度__ / 转化潜力__ / 合规稳定性__ / 总分__

结论：
- 推荐方案编号：<A/B/C>
- 推荐理由：
  1) ...
  2) ...
```

### 分镜硬规则

- 每套 3-5 镜，总时长 <=12 秒
- 每镜都要有：时长、画面描述、台词/旁白、卖点字幕、情绪标签、音效建议
- 前置镜头不露品牌，最后一镜必须品牌露出
- 人物仅轻互动，不主导叙事
- 禁止明显价格话术
- 禁止配料表主体表达

---

## 第4步：构建“推荐方案”即梦提示词

### 目标
把分镜脚本转成即梦可直接使用的生成提示。

### 模板（每套）

```
任务：根据以下分镜生成1条竖版短视频（仅推荐方案）
平台：<抖音/小红书/视频号>
时长：10-12秒
比例：9:16
分辨率：1080p
镜头脚本：<对应推荐方案分镜内容>

硬性要求：
1) 前置镜头不露品牌，最后一镜露品牌和商品
2) 不出现明确金额/价格表达
3) 不以配料表/成分列表为主体内容
4) 人物仅轻互动
5) 遵循对应平台内容规则
```

---

## 第5步：即梦生成（仅推荐方案）

### 执行规则

- 仅提交“推荐方案”1条生成任务
- 单条最多重试2次（共3次机会）
- 重试必须记录原因与改动点（开场钩子/镜头描述/CTA语气）

---

## 第6步：下载归档与质检

### 下载命名

- 推荐方案视频：`最终目录名-视频-推荐方案.mp4`

### 质检清单

- [ ] 推荐方案视频已下载至 `最终成片目录`
- [ ] 视频可播放，格式 `.mp4`
- [ ] 时长符合 10-12 秒
- [ ] 视频与推荐方案分镜一致
- [ ] 遵循对应平台规则，无平台错配
- [ ] 无明显金额/价格表达
- [ ] 无配料表主体表达

---
