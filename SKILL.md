# 七鲜商品内容策划（主图采集 + 步骤1-3）

用于视频生成前的前置内容策划，产出可直接用于 AI 工具的视频分镜脚本。

## 执行约束

- 先检查是否已提供商品主图；若未提供，必须先执行“主图采集”再进入后续步骤
- 必须基于用户输入生成三个产物：`切入点`、`创意结构`、`分镜脚本`
- 三个产物必须分别保存为独立文件，写入同一个商品目录
- 文件内容必须与用户输入强关联，不得输出与商品无关的泛化文案
- 视频脚本总时长必须 `<=12秒`
- 分镜脚本必须全程无人出镜（禁止人物、手部、身体局部、人物倒影/剪影）
- 若用户输入信息缺失，先用“最小可执行假设”补齐，并在文件末尾标注“假设项”

## 输入参数规范

执行前先整理以下输入（缺失可补默认值）：

```
商品名称:
skuId:
商品品类:
商品主图目录（可选，若为空则自动采集）:
核心卖点:
辅助卖点:
目标用户:
用户需求:
期望风格（可选）:
目标时长（默认12秒）:
输出根目录（默认 ~/7freshVedioGenerate）:
```

### 目录与文件规则（必须执行）

1. 创建商品目录：
   - `商品目录 = 输出根目录/商品名称-skuId`
   - 示例：`~/7freshVedioGenerate/芭乐樱樱茶-308443`
2. 主图文件命名（采集或用户提供后统一命名）：
   - `商品名称-skuId-1.jpg` ~ `商品名称-skuId-5.jpg`（通常5张）
3. 生成并保存 3 个文件：
   - `商品名称-skuId-切入点.md`
   - `商品名称-skuId-创意结构.md`
   - `商品名称-skuId-分镜脚本.md`
4. 主图与三个文档文件均保存到同一商品目录下

## 工作流程概览

```
┌──────────────────┐   ┌──────────────────┐   ┌──────────────────┐   ┌──────────────────┐
│ 0. 采集商品主图   │-> │ 1. 生成视频切入点 │-> │ 2. 设计创意结构   │-> │ 3. 生成分镜脚本   │
└──────────────────┘   └──────────────────┘   └──────────────────┘   └──────────────────┘
          │                       │                       │                       │
          v                       v                       v                       v
   无图则自动采集5张        卖点+用户需求拆解         输出传播主题与节奏        形成可执行镜头脚本
   有图则校验后复用          明确人群与消费场景         明确风格与价值表达        匹配配音、BGM、时长
```

## 第零步：获取商品主图（未提供时必须执行）

### 目标
从 `https://7fresh.m.jd.com` 获取指定商品主图并保存到商品目录，作为后续脚本参考素材。

### 触发条件

- 若用户已提供主图且通过质量检查：跳过采集，直接复用
- 若用户未提供主图，或提供主图不完整/不合格：执行采集流程

### 主图已提供时的跳过采集判定规则（必须执行）

满足以下条件才可跳过采集：

- 数量要求：至少 3 张，推荐 5 张
- 格式要求：JPG/JPEG（最终统一保存为 `.jpg`）
- 质量要求：单张图片建议 >100KB，且可正常打开
- 内容要求：至少覆盖 3 类画面（商品全貌 / 细节特写 / 场景或包装）

若任一条件不满足，执行以下策略：

- 数量不足 3 张：直接触发采集流程
- 数量为 3-4 张且质量合格：可复用现有图，但建议补采到 5 张
- 存在格式不统一：统一转存为 `.jpg` 并重命名
- 文件命名不规范：按规则自动重命名（见下方命名规则）
- 图片损坏或过小：剔除后重新采集替补

### 操作步骤

1. **打开七鲜移动端网站**
   - `https://7fresh.m.jd.com`

2. **搜索目标商品**
   - 通过商品名称搜索，或直接打开商品详情页 URL

3. **进入商详页并确认信息**
   - 核对商品名称
   - 获取并确认 `skuId`（通常在 URL 中，如 `skuId=308443`）

4. **下载商品主图**
   - 使用浏览器截图或图片下载方式获取主图
   - 下载全部主图（通常5张）

5. **统一命名并保存**
   - 保存目录：`~/7freshVedioGenerate/商品名称-skuId/`
   - 文件命名：`商品名称-skuId-1.jpg` ~ `商品名称-skuId-5.jpg`

### 主图自动重命名规则（复用用户已提供图片时）

- 按主图优先级排序后重命名：
  1) 商品全貌
  2) 原料/质地特写
  3) 卖点细节
  4) 场景图
  5) 包装或补充细节
- 重命名结果统一为：
  - `商品名称-skuId-1.jpg`
  - `商品名称-skuId-2.jpg`
  - `商品名称-skuId-3.jpg`
  - `商品名称-skuId-4.jpg`
  - `商品名称-skuId-5.jpg`

### 质量检查

下载完成后执行：

```bash
# 检查图片数量和大小
ls -lh ~/7freshVedioGenerate/商品名-skuId/

# 验证图片格式
file ~/7freshVedioGenerate/商品名-skuId/*.jpg
```

检查要点：

- [ ] 主图数量正确（通常5张）
- [ ] 图片格式为 JPG
- [ ] 图片大小合理（>100KB）
- [ ] 图片可以正常打开
- [ ] 主图命名符合 `商品名称-skuId-序号.jpg`

### 第零步保存要求（必须落盘）

- 主图文件必须保存到商品目录，与后续 3 个文档同目录
- 若采集失败，必须在回传结果中标记失败原因并停止后续步骤

## 第一步：根据卖点与用户需求生成视频切入点

### 目标
输入商品卖点和用户需求，产出“可情绪共鸣+可场景承接+可转化”的视频切入点。

### 操作步骤

1. **收集输入信息（必须完整）**
   - 商品名称、品类（果蔬/肉类/水产/预制菜）
   - 商品卖点（优先原料/工艺/口感/新鲜度/安全溯源）
   - 目标用户（家庭主妇/上班族/健身人群）
   - 用户需求（新鲜、健康、便捷、高性价比）

2. **完成信息拆解**
   - 提炼 `1个核心表达`（一句话说清“这款产品为什么值得拍”）
   - 提炼 `1个核心卖点 + 2-4个辅助卖点`
   - 匹配 `2-4个高频场景`（如晨跑后、野餐时、加班间隙、聚会）
   - 明确 `情绪主轴`（如心动/清爽/热恋/治愈）

3. **输出视频切入点（1-2个即可）**
   - 切入点A：情绪切入（示例：按心情解锁春日）
   - 切入点B：场景切入（示例：一天四个时刻，一杯刚好）
   - 每个切入点都要写清：卖点如何映射到情绪/场景

### 切入点输出格式

每个切入点统一输出：
```
切入点名称：
目标人群：
核心痛点：
核心表达（一句话）：
卖点-情绪映射：
  卖点1 -> 情绪1
  卖点2 -> 情绪2
  卖点3 -> 情绪3
场景映射：
  情绪1 -> 场景1
  情绪2 -> 场景2
建议开场话术（8-16字）：
收尾金句（15-25字，提问式优先）：
预期转化动作（评论互动/加购/下单）：
```

### 第一步保存要求（必须落盘）

- 文件名：`商品名称-skuId-切入点.md`
- 文件内容结构：
  - 输入信息摘要
  - 切入点 A（必填）
  - 切入点 B（可选）
  - 推荐采用切入点（必填，说明原因）
- 结尾补充：`假设项（如有）`

### 质量检查

- [ ] 切入点与卖点一一对应
- [ ] 切入点明确目标人群
- [ ] 切入点能在3秒内讲清价值
- [ ] 至少包含1条“卖点-情绪-场景”闭环
- [ ] 至少包含1个“新鲜/健康/便捷”关键词

## 第二步：根据切入点完成创意结构设计

### 目标
将选定切入点转化为可执行创意结构，明确“主题、风格、情绪节奏、转化动作”。

### 创意结构设计方法

1. **确定核心主题**
   - 例：`按心情解锁春日`
   - 主题必须同时覆盖“商品卖点+用户当下需求”

2. **确定视频风格**
   - 情绪氛围向：唯美特写 + 生活化场景 + 柔和配乐
   - 生活化转化向：真实使用 + 口语台词 + 明确 CTA
   - 质感高级向：干净布景 + 少字强视觉 + 节奏克制

3. **设计内容结构（推荐3-5镜，10-12秒）**
   - 首镜 吸睛开场：原料/成分唯美特写 + 第一情绪标签
   - 中段镜头 卖点递进：每镜头1个卖点 + 1个场景 + 1句口语台词 + 1句卖点文案
   - 尾镜 汇聚收尾：多场景拼接 + 品牌露出 + 提问式金句

4. **设置情绪节奏**
   - 0-3秒快速建立氛围
   - 3-9秒稳定输出卖点记忆点
   - 9-12秒完成品牌记忆与互动引导（如“你会选哪种心情”）

### 创意结构输出格式

```
核心主题：
目标人群：
视频风格：
核心价值主张：
情绪主轴（2-4个）：
内容结构（3-5镜）：
  1) 吸睛开场：
  2) 卖点镜头A：
  3) 卖点镜头B/汇聚收尾（3镜时可合并）：
  4) 卖点镜头C（4-5镜可选）：
  5) 汇聚收尾（4-5镜建议保留）：
每镜文案策略（必须填写）：
  - 对应用户需求：
  - 对应商品卖点：
  - 镜头卖点文案（10-22字）：
收尾转化动作（互动提问/购买引导）：
```

### 第二步保存要求（必须落盘）

- 文件名：`商品名称-skuId-创意结构.md`
- 文件内容结构：
  - 采用的切入点
  - 核心主题与价值主张
  - 风格定义
  - 3-5镜创意结构
  - 节奏分配（10-12秒）
  - 转化动作与 CTA 文案
- 结尾补充：`假设项（如有）`

### 质量检查

- [ ] 创意结构与切入点一致
- [ ] 结构可直接拆分为3-5镜头
- [ ] 核心价值主张可感知
- [ ] 适配短视频“短平快”传播特性
- [ ] 包含明确情绪标签与收尾金句
- [ ] 每镜已配置“用户需求+商品卖点+镜头文案”

## 第三步：生成分镜脚本

### 目标
基于创意结构生成“可直接投喂AI工具”的分镜脚本，确保画面、台词、音效可执行。
并确保全片仅包含商品、原料、器具与环境，不出现任何人物元素。

### 分镜结构

推荐 10-12 秒，3-5 镜头：

| 镜头 | 时长 | 内容 |
|------|------|------|
| 第1镜 | 0-2秒 | 原料唯美特写 + 情绪标签开场 |
| 第2镜 | 2-5秒 | 卖点A + 场景A |
| 第3镜 | 5-8秒 | 卖点B + 场景B（3镜方案可直接进入收尾） |
| 第4镜 | 8-10秒 | 卖点C + 场景C（4-5镜可选） |
| 第5镜 | 10-12秒 | 多场景汇聚 + 品牌露出 + 金句收尾（4-5镜建议） |

### 分镜脚本格式

每个镜头需包含：

```
镜头序号：
时长：
画面描述（含镜头运动/转场）：
场景信息（仅道具/环境细节）：
台词/旁白（口语化，8-20字）：
镜头卖点文案（10-22字，强绑定用户需求）：
卖点字幕（主标+副标）：
情绪标签（如心动/清爽/治愈）：
背景音乐建议：
环境音效建议：
```
将上面字段中的“场景信息”按以下规则执行：仅写道具状态、环境光影、物体运动，不写人物动作。

### 画面限制规则（硬性执行）

- 全片禁止出现：人物、手、脚、脸、身体局部、人物剪影、人物倒影
- 全片禁止出现：人与商品交互动作（如拿起、饮用、递交）
- 场景表达改为“物驱动”：
  - 用道具状态替代人物动作（如“杯壁水珠滑落”“瓶身轻微旋转”）
  - 用环境变化替代人物存在（如“窗帘微动”“阳光扫过桌面”）
- 若某镜头原设定依赖人物，必须重写为无人镜头后再输出

### 镜头卖点文案生成规则（必须遵循）

- 每个镜头必须单独生成 1 条卖点文案，不可复用同一句
- 文案必须同时体现：
  - 1 个明确商品卖点（如原料、口感、新鲜度、工艺）
  - 1 个明确用户需求（如清爽、便捷、健康、治愈感）
- 结构建议：`卖点事实 + 用户收益`
- 字数建议 10-22 字，避免空泛形容词堆砌
- 禁止只写品牌口号，必须对应该镜头画面内容

### 镜头卖点文案好/坏示例对照（给 qclaw）

**示例背景（芭乐樱樱茶）：**
- 镜头画面：晨光下带水珠的白芭乐茶饮置于长椅，瓶身轻微旋转
- 用户需求：清爽解腻、轻负担补水
- 商品卖点：四季香白芭乐，清甜不腻

**好示例（推荐）：**
- `四季香白芭乐清甜不腻，晨跑后这口更清爽。`
  - 命中卖点：四季香白芭乐、清甜不腻
  - 命中需求：晨跑后清爽补给
- `白芭乐果香轻盈不齁，运动后喝着刚刚好。`
  - 命中卖点：白芭乐果香、轻盈口感
  - 命中需求：运动后轻负担

**坏示例（避免）：**
- `春天就喝它，真的很好喝。`
  - 问题：无具体卖点、无用户需求、表达空泛
- `七鲜新品来了，快来买买买。`
  - 问题：只有促销口号，缺少镜头信息与价值说明
- `口感丰富层次多，品质非常高级。`
  - 问题：形容词堆砌，缺少可验证事实
- `一只手拧开瓶盖，清香马上出来。`
  - 问题：出现手部动作，不符合无人出镜限制

**快速判定标准（逐镜检查）：**
- 是否能指出“具体卖点是什么”
- 是否能回答“满足了用户什么需求”
- 是否与镜头画面强关联（而不是放到任何镜头都成立）

### 分镜脚本输出模板（建议直接使用）

```
项目：商品名称-skuId
总时长：XX秒（<=12秒）
镜头数：3-5镜

镜头1：
时长：
画面描述（含运镜/转场）：
场景信息（仅道具/环境细节）：
台词/旁白：
镜头卖点文案：
卖点字幕（主标+副标）：
情绪标签：
背景音乐建议：
环境音效建议：

镜头2：
...

镜头N：
...

收尾 CTA：
假设项（如有）：
```

### 脚本要求

- 文案简洁口语化，突出“新鲜、健康、便捷”
- 每镜头至少包含1个“卖点+场景+情绪”要素
- 每镜头必须有“镜头卖点文案”，且与用户需求强关联
- 台词避免硬广，优先“生活表达 + 情绪表达”
- 收尾必须有品牌名和一句提问/行动引导
- 总时长严格控制在12秒内（建议10-12秒）
- 分镜脚本文案总字数不超过1900字（硬性限制）
- 全片禁止人物出镜（包含手部/倒影/剪影）

### 质量检查

- [ ] 总时长不超过12秒（建议10-12秒）
- [ ] 包含3-5个镜头
- [ ] 每个镜头有台词、镜头卖点文案、卖点字幕
- [ ] 有背景音乐建议
- [ ] 有情绪标签与环境音效建议
- [ ] 逻辑为“吸睛→卖点递进→场景承接→品牌收尾→转化”
- [ ] 分镜脚本文案总字数 <= 1900
- [ ] 全片无人出镜（含手部/倒影/剪影）
- [ ] 已保存为 `商品名称-skuId-分镜脚本.md`

### 第三步保存要求（必须落盘）

- 文件名：`商品名称-skuId-分镜脚本.md`
- 文件编码：UTF-8
- 内容要求：可直接复制到视频生成工具使用
